void pros::Mutex::lock ( )

Takes and locks a mutex, waiting for up to TIMEOUT_MAX milliseconds.

Effectively equivalent to calling pros::Mutex::take with TIMEOUT_MAX as the parameter.

Conforms to named requirment BasicLockable

See also
https://en.cppreference.com/w/cpp/named_req/BasicLockable
Note
Consider using a std::unique_lock, std::lock_guard, or std::scoped_lock instead of interacting with the Mutex directly.
Exceptions
std::system_errorMutex could not be locked within TIMEOUT_MAX milliseconds. see errno for details.
